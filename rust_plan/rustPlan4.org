Installation requires root privileges, so dnf operations need sudo handling like apt.

Error handling should catch common DNF errors: package not found, dependency conflicts, and repository issues. Parse stderr to provide helpful messages.

For Nexus, DNF support enables Fedora and RHEL users. The implementation follows the same patterns as your apt backend but with dnf-specific commands and parsing.

** TODO Day 166: Binary Size Optimization

*** Reading

    Rust binary size optimization (https://github.com/johnthagen/min-sized-rust)
    Cargo profile options

*** Summary

Smaller binaries distribute more easily and load faster. Release builds are already optimized, but further reductions are possible.

Link-time optimization (LTO) enables cross-module optimization. Set lto = true in release profile.

Stripping symbols removes debug information. Set strip = true or use the strip command.

Single codegen unit allows more optimization at cost of compile time. Set codegen-units = 1.

Panic abort replaces unwinding with immediate termination. Set panic = "abort" for smaller but less graceful panics.

Opt-level 'z' optimizes for size over speed. Usually 's' or default is better for balanced optimization.

For Nexus, reasonable binary size matters for distribution. A few megabytes is acceptable; tens is not.
