Directory Setup

Bash

# Create project directory
mkdir -p ~/projects/nexus
cd ~/projects/nexus

# Create the flake
cat > flake.nix << 'EOF'
# [paste the flake.nix content above]
EOF

# Create .envrc for direnv (auto-loads environment)
echo 'use flake' > .envrc

# Allow direnv (one-time)
direnv allow

# Or enter manually without direnv
nix develop

.envrc (for automatic environment loading)

Bash

How to Use
Initial Setup (One Time)

Bash

# 1. Create project directory
mkdir -p ~/projects/nexus
cd ~/projects/nexus

# 2. Save the flake.nix (copy from above)
# ... save flake.nix ...

# 3. Initialize git (required for flakes)
git init
git add flake.nix

# 4. Enter the development environment
nix develop

# 5. Create Rust project
cargo new . --name nexus
# Or if directory exists:
cargo init --name nexus

# 6. Verify everything works
cargo check
rustc --version
cargo clippy --version
rust-analyzer --version

Daily Development

Bash

# Option A: Manual entry
cd ~/projects/nexus
nix develop

# Option B: With direnv (automatic)
cd ~/projects/nexus
# Environment loads automatically!

Common Commands

Bash

# Quick aliases (available after entering environment)
cc          # cargo check (fast syntax check)
cb          # cargo build
cr          # cargo run
ct          # cargo test
cl          # cargo clippy
cf          # cargo fmt
cw          # cargo watch -x check (auto-rebuild)
cdoc        # cargo doc --open

# Full commands
cargo build --release           # Release build
cargo test -- --nocapture       # Tests with output
cargo clippy -- -D warnings     # Strict linting
cargo tarpaulin --out Html      # Code coverage
cargo flamegraph                # Performance profiling
cargo audit                     # Security check
cargo outdated                  # Check for updates
cargo bloat --release           # Binary size analysis

Updating the Flake

Bash

# Update all inputs (Rust version, nixpkgs, etc.)
nix flake update

# Update specific input
nix flake lock --update-input rust-overlay
