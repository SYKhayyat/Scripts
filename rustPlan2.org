This is a 10-week structured plan to learn Rust while rebuilding LiNix (a declarative package manager) as a term project. Designed for a Java/Spring Boot student familiar with Data Structures.

- Map Rust concepts to familiar Java paradigms

*** To-Do**:

   let a = 42;
   let b = a;
   println!("{}", a);
   
   // C: Works (explicit clone)
   let a = String::from("git");
   let b = a.clone();

  ─────────────────────────────────────────────
  class Foo { }                 struct Foo { }
  interface Bar { }             trait Bar { }
  implements Bar                impl Bar for Foo
  extends Parent                (use composition)
  new Foo()                     Foo { field: value } or Foo::new()
  this                          self
  null                          None (via Option<T>)
  try/catch                     Result<T, E> + ?
  ArrayList<T>                  Vec<T>
  HashMap<K,V>                  HashMap<String, Vec<String>>  // Note nested types
  String                        String (owned) / &str (borrowed)
  final                         (default — immutable is safe)
  (mutable)                     let mut
  System.out.println()          println!()
  for (T x : list)              for x in &list
  lambda: x -> x+1              closure: |x| x+1
  throws Exception              -> Result<T, E>
  @Override                     (automatic when impl trait)
  package / import              mod / use
  public                        pub
  private                       (default — security first!)
  ```

*** Common Traps to Avoid
1. **Don't fight the borrow checker** — if it says no, you have a design issue. Clone early, optimize later.
2. **Don't use `unwrap()` in production code** — use `?` or `match`. Every `unwrap()` is a potential crash.
3. **Don't over-engineer lifetimes** — if adding `'a` everywhere, you should clone instead.
4. **Don't skip `cargo clippy`** — it teaches idiomatic Rust better than any book.
5. **Read compiler errors top-to-bottom** — they're the best in any language and tell you how to fix them.
